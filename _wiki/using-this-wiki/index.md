# Using This Wiki

Welcome! This section is your complete guide to running, customizing, and growing this knowledge base. You do **not** need prior experience with Jekyll, GitHub Pages, or Markdown. Follow these pages in order, or jump to the topic you need.

## Where to start

1. [Adding Content](/wiki/using-this-wiki/adding-content/) — create pages and categories.
2. [Editing Settings](/wiki/using-this-wiki/editing-settings/) — change the site title, colors, navigation, and more.
3. [Managing Repo](/wiki/using-this-wiki/managing-repo/) — GitHub workflow, previewing, and troubleshooting.
4. [Using This Wiki Fully](/wiki/using-this-wiki/fully/) — complete reference, front matter cheat sheet, and pro tips.

## What this site does

- Turns Markdown files into a fast, searchable documentation website.
- Supports dark and light themes with a single click.
- Works on phones, tablets, and desktops.
- Builds and deploys automatically when you push to GitHub.

## How to preview

You can preview changes locally before pushing:

```bash
bundle install
bundle exec jekyll serve
```

Then open `http://localhost:4000/<baseurl>/` in your browser.

## Need help?

Each manual page includes copy-paste templates and step-by-step instructions. If something breaks, check [Managing Repo](/wiki/using-this-wiki/managing-repo/) for troubleshooting tips.
